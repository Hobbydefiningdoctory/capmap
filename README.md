# capman

**Capability Manifest Engine** — Define what your app can do, so AI agents can discover, match, and invoke it.

`capman` lets you declare your app's capabilities in a simple config file and generates a structured manifest that AI agents can use to understand your API, navigate your UI, and interact with your data — all with privacy and resolver rules baked in.

---

## Features

- Declare app capabilities with descriptions, examples, params, and resolvers
- Generate a structured `manifest.json` from a simple config file
- Match natural-language queries to capabilities with confidence scoring
- Resolve matched capabilities to real API calls or navigation actions
- CLI for init, generate, validate, and inspect workflows
- Written in TypeScript with full type safety

## CLI Usage

```bash
node bin/capman.js <command> [options]
```

### Commands

| Command    | Description                                      |
|------------|--------------------------------------------------|
| `init`     | Create a starter `capman.config.js`              |
| `generate` | Generate `manifest.json` from config             |
| `validate` | Validate an existing `manifest.json`             |
| `inspect`  | Print all capabilities in the manifest           |

### Options

| Option       | Default             | Description                  |
|--------------|---------------------|------------------------------|
| `--config`   | `capman.config.js`  | Path to config file          |
| `--out`      | `manifest.json`     | Output path for manifest     |
| `--manifest` | `manifest.json`     | Manifest file to read        |

### Quick Start

```bash
# 1. Create a starter config
node bin/capman.js init

# 2. Edit capman.config.js with your app's capabilities

# 3. Generate the manifest
node bin/capman.js generate

# 4. Validate it
node bin/capman.js validate

# 5. Inspect the registered capabilities
node bin/capman.js inspect
```

## Programmatic Usage

```ts
import { generate, match, resolve } from './src/index'
import type { CapmanConfig } from './src/types'

const config: CapmanConfig = {
  app: 'my-app',
  baseUrl: 'https://api.my-app.com',
  capabilities: [
    {
      id: 'get_resource',
      name: 'Get a resource',
      description: 'Fetch a specific resource by name, ID, or filter.',
      examples: ['Find resource by ID', 'Look up resource by name'],
      params: [
        {
          name: 'resource_id',
          description: 'The ID or name of the resource',
          required: true,
          source: 'user_query',
        },
      ],
      returns: ['resource', 'metadata'],
      resolver: {
        type: 'api',
        endpoints: [{ method: 'GET', path: '/resources/{resource_id}' }],
      },
      privacy: { level: 'public' },
    },
  ],
}

// Generate manifest
const manifest = generate(config)

// Match a natural-language query
const result = match('Find resource by ID', manifest)
console.log(result.capability?.id)   // get_resource
console.log(result.confidence)       // 100

// Resolve the match to an API call
const resolution = await resolve(
  result,
  { resource_id: '42' },
  { baseUrl: 'https://api.my-app.com', dryRun: true }
)
console.log(resolution.apiCalls)     // [{ method: 'GET', url: '...' }]
```

## Config File

`capman.config.js` defines your app's capabilities:

```js
module.exports = {
  app: 'your-app-name',
  baseUrl: 'https://api.your-app.com',
  capabilities: [
    {
      id: 'navigate_to_screen',
      name: 'Navigate to a screen',
      description: 'Route the user to a specific page or section in the app.',
      examples: ['Take me to the dashboard', 'Open settings'],
      params: [
        {
          name: 'destination',
          description: 'The screen or page to navigate to',
          required: true,
          source: 'user_query',
        },
      ],
      returns: ['deep_link'],
      resolver: { type: 'nav', destination: '{destination}' },
      privacy: { level: 'public' },
    },
  ],
}
```

### Privacy Levels

| Level        | Description                                      |
|--------------|--------------------------------------------------|
| `public`     | No authentication required                       |
| `user_owned` | Scoped to the currently authenticated user       |
| `private`    | Internal use only                                |

### Resolver Types

| Type  | Description                              |
|-------|------------------------------------------|
| `api` | Maps to one or more HTTP API endpoints   |
| `nav` | Deep-links to a screen or route          |
